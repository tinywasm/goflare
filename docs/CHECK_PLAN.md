# goflare вЂ” Bug Fix Plan

## Bug 1: Edge compiler uses wrong input file

### Error

```
worker build failed: compileSync build failed: exit status 1
package edge/client.go is not in std
```

### Root cause

`goflare.go` creates `edgeCompiler` via `client.New()` without overriding two
hardcoded defaults from `tinywasm/client`:

| Field | Default (WasmClient) | Required for edge |
|-------|---------------------|-------------------|
| `MainInputFile` | `"client.go"` | `"main.go"` |
| `OutputName` | `"client"` | `"edge"` |

TinyGo is invoked as `tinygo build edge/client.go` вЂ” which does not exist.

Secondary: `wasm.go` rename logic computes `filepath.Base(Entry)+".wasm" = "edge.wasm"`
but the compiler produced `client.wasm`. Rename silently skips в†’ deployment finds no `edge.wasm`.

### Fix вЂ” `goflare/goflare.go`

After `client.New(...)`, add:

```go
// Edge functions use main.go (not client.go, which is the frontend default).
// OutputName "edge" makes TinyGo produce edge.wasm directly вЂ” no rename needed.
edgeCompiler.SetMainInputFile("main.go")
edgeCompiler.SetOutputName("edge")
```

### Simplification вЂ” `goflare/wasm.go`

With `OutputName="edge"` the rename block is dead code вЂ” remove it:

```go
func (g *Goflare) generateWasmFile() error {
    return g.edgeCompiler.RecompileMainWasm()
}
```

## Bug 2: Unnecessary copy `web/public/ в†’ .build/dist/`

### Problem

`buildPages()` copies `PUBLIC_DIR` в†’ `.build/dist/` and `DeployPages` reads from `.build/dist/`.
The copy is a no-op transformation: `copyDir` does a literal byte copy with no modifications.

`web/public/` is already the complete deployable directory after build:
- `client.wasm` вЂ” compiled there by `browserCompiler` (`OutputDir = cfg.PublicDir`)
- `script.js`, `style.css` вЂ” generated there by `assetmin`
- `index.html`, static assets вЂ” already there

The copy exists only because `DeployPages` hardcoded `dist/` as the source.
It wastes I/O (especially with `.wasm` files) and adds a confusing indirection.

### Fix вЂ” `goflare/cloudflare.go`

`DeployPages` reads from `cfg.PublicDir` directly:

```go
// Before:
distDir := filepath.Join(g.Config.OutputDir, "dist")

// After:
distDir := g.Config.PublicDir
```

### Fix вЂ” `goflare/build.go`

Remove the copy step from `buildPages()`. Steps 1вЂ“3 remain, step 4 is deleted:

```go
func (g *Goflare) buildPages() error {
    if _, err := os.Stat(g.Config.PublicDir); os.IsNotExist(err) {
        return fmt.Errorf("public dir does not exist: %s", g.Config.PublicDir)
    }

    frontEntry := filepath.Join("web", "client.go")
    if _, err := os.Stat(frontEntry); err == nil {
        if g.browserCompiler == nil {
            return fmt.Errorf("frontend compiler not initialized (browserCompiler is nil)")
        }
        g.Logger("compiling frontend WASM: web/client.go в†’", g.Config.PublicDir)
        if err := g.browserCompiler.Compile(); err != nil {
            return fmt.Errorf("frontend WASM compilation failed: %w", err)
        }
    }

    if g.assetMin != nil {
        g.Logger("generating assets: script.js, style.css в†’", g.Config.PublicDir)
        g.assetMin.SetBuildOnDisk(true)
    }

    return nil
}
```

Also remove the now-unused `copyDir` and `copyFile` helpers from `build.go`.

## Tests

### Design principles

- One concern per file вЂ” each file tests one behavior contract
- Shared setup via `testEnv` in `setup_test.go` вЂ” no repeated `os.MkdirAll` / file creation in individual tests
- `helpers_test.go` keeps `TempDir()` and `MockHTTPServer()` as low-level primitives
- `testEnv` is a higher-level fixture built on top of those primitives

### `tests/setup_test.go` вЂ” shared fixture (replaces inline dir setup in every test)

```go
package goflare_test

import (
    "os"
    "path/filepath"
    "testing"
)

// testEnv is a pre-wired temporary project layout:
//   <root>/
//     web/public/       в†ђ PublicDir
//       index.html
//     .build/           в†ђ OutputDir
type testEnv struct {
    Root      string
    PublicDir string
    OutputDir string
    cleanup   func()
}

func newTestEnv(t *testing.T) *testEnv {
    t.Helper()
    dir, cleanup, err := TempDir()
    if err != nil {
        t.Fatalf("testEnv: %v", err)
    }
    pub := filepath.Join(dir, "web", "public")
    out := filepath.Join(dir, ".build")
    os.MkdirAll(pub, 0755)
    os.MkdirAll(out, 0755)
    os.WriteFile(filepath.Join(pub, "index.html"), []byte("<h1>test</h1>"), 0644)
    return &testEnv{Root: dir, PublicDir: pub, OutputDir: out, cleanup: cleanup}
}

func (e *testEnv) Close() { e.cleanup() }

// writePublic writes a file into PublicDir.
func (e *testEnv) writePublic(name, content string) {
    os.WriteFile(filepath.Join(e.PublicDir, name), []byte(content), 0644)
}

// writeOutput writes a file into OutputDir (simulates pre-built artifacts).
func (e *testEnv) writeOutput(name, content string) {
    os.WriteFile(filepath.Join(e.OutputDir, name), []byte(content), 0644)
}
```

### TinyGo installation вЂ” content that must appear in Requirements

The README must document `tinywasm/tinygo` as the automated installer so developers
and LLMs do not assume a manual TinyGo setup is required.

```
## Requirements

- Go 1.25.2+
- TinyGo вЂ” installed automatically by `goflare build` via `tinywasm/tinygo`

`goflare` calls `tinygo.EnsureInstalled()` before each Worker build.
If TinyGo is already in `PATH` at the correct version, nothing happens.
If it is missing or outdated, it is downloaded and installed to the local cache automatically.

To manage TinyGo independently in your own tooling:

    import "github.com/tinywasm/tinygo"

    // Ensure correct version is installed (idempotent)
    tinygo.EnsureInstalled()

    // Get env slice with TINYGOROOT + PATH вЂ” pass to exec.Command
    env := tinygo.GetEnv()
```
