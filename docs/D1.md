# D1 — Cloudflare D1 ORM Adapter

`github.com/tinywasm/goflare/d1` connects a Cloudflare D1 SQLite database to `tinywasm/orm` from within a Worker or Pages Function edge handler.

## Public API

```go
// New opens the named D1 binding and returns an *orm.DB.
// WASM only — reads the binding from context.env.
// Returns ErrDatabaseNotFound if the binding is absent from the Worker env.
func New(bindingName string) (*orm.DB, error)

// NewDirect opens a D1 database via the Cloudflare REST API and returns an *orm.DB.
// Host only (!wasm) — uses the same sqlt.NewCompiler() as New(), so SQL generation
// is identical. Intended for integration tests; not for production edge code.
func NewDirect(token, accountID, databaseID string) (*orm.DB, error)

var ErrDatabaseNotFound error
```

`*orm.DB` exposes the standard tinywasm ORM methods: `Create`, `First`, `Save`, `Delete`, `CreateTable`, `DropTable`, `Close`.

## Usage

```go
//go:build wasm

package contact

import "github.com/tinywasm/goflare/d1"

func saveSubmission(form ContactForm) error {
    db, err := d1.New("DB") // "DB" matches the binding name in wrangler.toml
    if err != nil {
        return err
    }
    defer db.Close()

    if err := db.CreateTable(&ContactSubmission{}); err != nil {
        return err
    }
    return db.Create(&ContactSubmission{
        Nombre:  form.Nombre,
        Email:   form.Email,
        Mensaje: form.Mensaje,
    })
}
```

## Model setup

The model struct must implement `fmt.Model`. Use the `// ormc:form` directive and run `ormc` — it generates `ModelName()`, `Schema()`, `Pointers()`, and `Validate()` automatically:

```go
// ormc:form
type ContactSubmission struct {
    ID      int64
    Nombre  string
    Email   string
    Mensaje string
}
```

```bash
ormc
```

`ID int64` is auto-detected as primary key (auto-increment). No extra tags needed.

## wrangler.toml

```toml
[[d1_databases]]
binding      = "DB"
database_name = "my-db"
database_id   = "your-database-id"
```

## Integration testing

`NewDirect` permite testear el mismo código ORM que corre en el edge, pero desde el host, contra la API REST de D1. El compilador SQL es `sqlt.NewCompiler()` en ambos casos — los mismos errores que aparecerían en producción se detectan en el test.

```bash
CLOUDFLARE_API_TOKEN=cfat_... CLOUDFLARE_ACCOUNT_ID=xxx D1_DATABASE_ID=yyy \
  go test -tags=integration -run TestD1Integration ./d1/ -v
```

Sin credenciales el test se omite (`t.Skip`) — nunca falla en rojo.

Para CI, ver [CI_D1_SECRETS.md](CI_D1_SECRETS.md) — instrucciones paso a paso con capturas de pantalla.

## Constraints

- `d1` only compiles for `GOOS=js GOARCH=wasm` — all files carry `//go:build wasm`.
- Do **not** import `d1` from `web/server.go` or any non-wasm file.
- No `database/sql`. D1 is accessed exclusively via the D1 JS API.
- No stdlib `fmt`, `errors`, `strings` — use `github.com/tinywasm/fmt`.

## Architecture

```
orm.DB
 ├── d1.adapter        (orm.Executor, //go:build wasm)    — calls D1 JS API via AwaitPromise
 ├── d1.directAdapter  (orm.Executor, //go:build !wasm)   — calls D1 REST API via net/http
 └── sqlt.Compiler     (orm.Compiler, both)               — generates SQLite-dialect SQL
```

| Operation | D1 JS call |
|---|---|
| `Exec` | `db.prepare(sql).bind(...args).run()` |
| `QueryRow` | `db.prepare(sql).bind(...args).first()` |
| `Query` | `db.prepare(sql).bind(...args).raw({columnNames:true})` |

The D1 binding is read from `js.Global().Get("context").Get("env").Get(bindingName)`.
